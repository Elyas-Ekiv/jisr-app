=== SYSTEM REQUIREMENTS ===
- Free disk space: Minimum 2GB for node_modules
- Memory: 512MB-1GB RAM minimum
- Database: PostgreSQL 12+ (Prisma requirement)
- Max file upload size: Configure in:
  - Express middleware: app.use(express.json({ limit: '50mb' }))
  - multer: limits: { fileSize: 50 * 1024 * 1024 }

=== DATABASE CONFIGURATION ===
- PostgreSQL connection required
- Prisma ORM configuration in: backend/prisma/schema.prisma
- Setup DATABASE_URL in .env:
  DATABASE_URL="postgresql://user:password@host:5432/jisr_db"
- Run migrations: npm run prisma:migrate (before first deployment)
- Seed database: npm run prisma:seed (optional)

=== CPANEL DEPLOYMENT STEPS ===

1. File Upload:
   - Upload project files via FTP/SFTP
   - Exclude: node_modules/, dist/, uploads/media/, .env, .git/
   - Include: package.json, package-lock.json, .gitignore, .env.example

2. Install Dependencies:
   cd /path/to/project
   npm ci (use 'ci' instead of 'install' for production)

3. Frontend Build:
   npm run build
   (This creates dist/ folder with optimized static files)

4. Backend Setup:
   cd backend
   npm ci
   npm run prisma:generate
   npm run prisma:migrate
   npm run build

5. Environment Setup:
   - Create .env in root with frontend config
   - Create backend/.env with backend config
   - Ensure .env files are NOT in version control

6. Service Configuration:
   - Use cPanel's Node.js app to manage backend process
   - Or use PM2 for process management:
     npm install -g pm2
     pm2 start backend/dist/server.js --name "jisr-backend"
     pm2 save

7. Reverse Proxy Setup (if needed):
   - Configure Apache/Nginx to proxy requests to Node.js backend
   - Frontend serves from dist/ (static files)
   - Backend API proxied to Node.js process

8. SSL Certificate:
   - Enable AutoSSL in cPanel
   - Use certbot if manual setup needed
   - Ensure HTTPS for payment gateway

=== SECURITY CONSIDERATIONS ===
- Set appropriate file permissions: 755 for folders, 644 for files
- Keep .env files outside web root
- Use strong JWT_SECRET (minimum 32 characters)
- Enable helmet middleware for security headers
- Configure CORS properly for production domain
- Rate limiting enabled (express-rate-limit)
- Validate all inputs (express-validator)
- Hash passwords with bcryptjs (10 rounds minimum)

=== PERFORMANCE OPTIMIZATION ===
- Enable gzip compression in Express
- Use sharp for image optimization
- Implement database indexing in Prisma
- Cache static files with 1 year expiry
- Use CDN for media files if possible
- Enable clustering/worker processes

=== MAINTENANCE & MONITORING ===
- Regular database backups (PostgreSQL)
- Monitor disk space for uploads/
- Check Node.js logs: .../logs/
- Monitor PM2 process: pm2 monit
- Regular security updates: npm audit fix
- Database migrations on updates

=== BUILD COMMANDS ===
Frontend:
  npm run build        # Production build (creates dist/)
  npm run lint         # Check code quality

Backend:
  npm run build        # Compile TypeScript to JavaScript
  npm run prisma:generate  # Generate Prisma client
  npm run start        # Start production server

Combined Deploy:
  npm run build && cd backend && npm run build && npm start

=== TROUBLESHOOTING ===
- Module not found: Run npm ci and npm run prisma:generate
- Port already in use: Change PORT in .env
- Database connection error: Verify DATABASE_URL and PostgreSQL running
- File upload fails: Check UPLOAD_DIR permissions and MAX_FILE_SIZE
- Payment gateway error: Verify Thawani keys in .env
